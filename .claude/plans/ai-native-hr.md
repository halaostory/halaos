## 核心理念

### "AI 不是功能，AI 就是产品本身"

传统 HR 系统：菜单 → 表单 → 提交 → 等审批 → 查结果
AI-Native HR：**AI 自动感知 → 自动执行 → 通知确认 → 异常才需人工**

### 黄仁勋 Vision 在 AIGoNHR 的落地

黄仁勋："未来软件公司不再卖许可证，而是出租专业智能体替客户完成工作，核心营收方式变成租赁 Tokens。"

**AIGoNHR 已经走在这条路上：**
- 我们不卖 per-seat 许可证 → 卖 Token 包（PHP 99-6,999）
- 不是给你 HR 软件 → 是给你 8 个 HR 专家智能体帮你干活
- 员工不操作系统 → 智能体替员工操作系统

**更进一步的 Agentic 转型：**
1. **AIGoNHR = HR Department as a Service** — 不是卖软件给你的 HR 部门用，而是 AIGoNHR 就是你的 HR 部门
2. **Hybrid Agent Model** — 自有模型（知识库 + 规则引擎）处理 70% 日常任务（免费），租赁 LLM（Claude/GPT）处理 30% 复杂任务（Token 收费）
3. **Agent Specialization Pricing** — 复杂 Agent（薪资计算、合规审查）定价更高，正如人类专家咨询费更贵
4. **Agent Output = Business Deliverable** — Agent 不只是回答问题，而是产出可交付成果：薪资单 PDF、政府报表、在职证明、离职清算报告

## 一、零操作 HR（Zero-Operation）

### 1.1 智能打卡 —— 走进办公室就完成
**功能**: 员工手机连上公司 WiFi / 进入地理围栏 → 自动打卡
**懒人程度**: 5/5 | **惊艳程度**: 4/5 | **复杂度**: 3/5

**技术实现**:
- PWA Service Worker 监听 geolocation
- 检测进入围栏 → POST `/api/v1/attendance/auto-checkin`
- Redis 去重窗口 30 分钟
- 置信度不够 → 推送通知确认

### 1.2 缺勤自动转假 —— AI 帮你请假
**功能**: 10AM 未打卡 → AI 通知 → 员工点 [请病假] → 自动提交
**懒人程度**: 5/5 | **惊艳程度**: 5/5 | **复杂度**: 4/5

**技术实现**:
- Worker 10AM 扫描无打卡员工
- 交互式通知：[请病假] [请事假] [在路上] [忽略]
- 无响应 → 标记旷工通知经理

## 商业模型进化（黄仁勋 Token 经济学）

### 从 "SaaS 许可证" → "AI Agent 租赁"

| 维度 | 传统 HR SaaS | AIGoNHR Agentic |
|------|-------------|-----------------|
| 定价 | per-seat PHP 200-500/月 | Token 包 PHP 99-6,999 |
| 交付物 | 软件界面 | 专家智能体 + 工作成果 |
| 成本结构 | 固定（服务器） | 变动（LLM API） |
| 用户体验 | 操作软件 | 对话 + 自动化 |
| 价值衡量 | 功能数量 | 完成的 HR 任务数 |
| 规模效应 | 线性（per-seat） | 超线性（Token 越用越多） |

### Agent 分层定价（专业度 = 定价）

| Agent 层级 | 示例 | Token 倍率 | 类比 |
|-----------|------|-----------|------|
| 基础查询 | 假期余额、考勤状态 | 1.0x | 前台接待 |
| 标准操作 | 请假、打卡、报销 | 1.2x | HR 专员 |
| 专业分析 | 薪资计算、合规检查 | 1.5x | HR 经理 |
| 高级预测 | 离职预测、异常检测 | 2.0x | HR 总监 |
| 自动化执行 | 发薪、入职、清算 | 2.5x | HR VP |

### 免费 vs 收费边界

| 免费（规则引擎，无 LLM） | 收费（LLM，消耗 Token） |
|-------------------------|----------------------|
| 自动打卡（地理围栏规则） | AI 缺勤原因分析 |
| 定时合规提醒（模板文本） | AI 生成合规报告 |
| 离职风险分数（SQL 计算） | AI 生成风险解释文本 |
| 表单格式校验 | AI 智能预填表单 |
| 文档过期提醒 | AI 文档内容分类 |

**关键洞察**: 70% HR 日常可由规则引擎免费处理（吸引用户），30% 需要 LLM 的智能分析/生成（Token 收入）。这就是黄仁勋说的"自有模型 + 租赁模型混合"。

## 架构要点

1. **hr_events 驱动一切** — 所有 AI 自动操作通过事件系统执行和审计
2. **置信度分层** — 高置信度静默执行，低置信度询问，极低不执行
3. **人类覆盖** — 法律/财务操作必须人工确认，其他可自动
4. **混合模型** — 规则引擎处理日常（免费），LLM 处理复杂（Token）
5. **审计追踪** — 每个 AI 决策完整记录在 ai_audit_log
